How I build a single page with video background

2017-04-07

Why I am talking about single page?

These days, most requirements from my works were about building specific components of website, it may sound fashion while developers could tell people he/she is working on some components stuff, but at same time, I couldn’t help but thinking about am I really getting better through this process.
Well, I guess I finally have the chance to find out the answer of this question.

New Task

Recently, I got a new task which required me to build a career page for our company, it’s simple, it’s static, it’s just a single page, and it also needs a video background on it’s banner, nothing sounds complex, however, I was excited about this task, I finally had a chance to build a whole new page from scratch for my job, and since there were several new knowledge I have learned during these period, I knew I can do it in a better way, and I was keen to find out how I would go through the whole process.

Requirements

There are several requirements of this task.

  • Background video auto played with position centered in any width desktop version.
  • Grids different sequence in desktop and mobile version. (Fortunately we have Flex.)
  • Hidden content, can be seen by clickable button. (Build a modal by JS)
  • And of course, full responsive. (I prefer build CSS in Mobile First Design and BEM these days.)

Approaches

First of all, I took a step back and thought about whole of it for a while,
I wanted this to be very simple, maintainable, and readable,
then I started to ask my self, what do I need to do first?

A Great Structure Template

For sure, I would definitely need a template as a reference to build the file structure, so I searched it on github and decided to take this repo as reference.
Why did I choose it? Well..

  • Simple structure
  • Just html
  • No Gulp (No need to use Gulp since I use Prepros to compile less.)

javascript

In this project, I tried to use javascript as less as possible,
just wanted to take it as a challenge for myself.
however, there were 2 different parts still required me

01. Modal

I knew there are several ways to build modal by pure css, but if I built it in that way, every time we add one more job position, I will need to add one more Modal division as well, imaging if we have 10 positions, how messy this html is gonna be….
Therefore, my approach was building ONE <div>, injecting different job description when triggering modal.

02. Layout

Since I made the position of content is absolute, the height of outer div became zero, but my approach was to let both of them have same height, I tried to find a solution to solve it in CSS way, however, I knew this might be possible, but is it worthy to spend extra time for this while you could build it in a short time with javascript, I guessed the answer is NO.

Conclusion

As I mentioned at beginning, there is nothing complex in this project and article, the reason why I wrote down this developing process is because I wanted to know what I were thinking while I need to make decision.
Wanted it simple? How?
Solve UI tasks? what are my solutions?
Using javascript instead of CSS, why? Am I reasonable?
I wanted to force myself starting thinking, and inspecting myself to see if I am being reasonable, logic, furthermore, building experience for becoming a senior Front End developer.

Update–2017/05/08

Since this site has been built and uploaded, there are also several things has been changed according to the requirements of Performance, I also learned few things through this process.

01. Network Throttle Emulator:

This emulator is in Network section in Dev Tool, which can help us inspect the website in different networking speed situations, due to the result of this emulator, we optimized the size of images and video.

02. Page should work without Javascript:

I guess this one is optional, depending on the marketing strategy and scale of the website, however, since this job page is so simple, nothing complicated, obviously there is no reason we should skip it, and actually this requirement let me notice that I should use <button> or <a> as clickable elements instead of <div>, shame on me, I should have known this since it seems so basic.

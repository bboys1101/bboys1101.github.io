Working Note - Mobile First CSS

2017-01-12

Requirements of Current Project.

After building E-Mail Generator, I am about to jump into the main project that our team are currently building together. Mainly we are responsible for creating and testing the modularized html components of each sites, which is constructed by template engine - Jade, and using scss and BEM naming convention to edit the style sheet.

Before jumping into this project, I was aware of that Mobile First CSS is the main concept of the process for building the site here. As a web developer, mobile first is a term that I am so familiar with, because everyone talks about it all the time. However, I’ve found that I actually don’t have a clear concept about Mobile First CSS, so I spent some time to do a deep learning about it, the following is my technical note about it.

What is Mobile First CSS?

1. Mobile First Design vs Mobile First CSS.

Actually, they have nothing to do with each other, you can architect you style sheet in Mobile First method even the site isn’t Mobile First Design, like mrmrs mentioned in his article,

Mobile-first CSS has to do with how you architect your code and media queries, as well as how browsers parse your CSS.

With that concept in mind, I can develop our css code from mobile version instead of thinking desktop first like I used to do before.

Pros:

1. Browser will only parse the necessary code.

As the example code above, we build the style attribute from mobile, so every code inside the media queries is for defining the wider device, like tablet and desktop, then browser will not parse these code if we are viewing the site by mobile. However, in desktop first CSS, the browser will parse the whole code for desktop no matter it is necessary or not.

2. Reducing the code.

Since the default value width forul and div is "auto", it means we don’t have to specify it at mobile version, only need to write down different value in media queries for wider device, so we actually can write less code compare to Desktop First.

3. Avoid downloading unnecessary image.

According to this research done by Tim Kadlec, it reveals that the requests still happen EVEN we hide image by setting display: nonein media queries for mobile device, this would cause browser has lower performance due to download unnecessary images. However, in Mobile First CSS design, we will not download the background-img until the width of browser is larger than break point, which makes mobile browsers could have better performance.

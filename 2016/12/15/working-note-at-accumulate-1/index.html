What!? JSON not updating!

I have been working on E-Mail template generator with Nunjucks and Gulp recently. In the gulpfile.js, I built a task which can allow us to inject data object to Nunjucks components, so it can populate content from JSON file, however, I noticed that the content of static html wasn’t updated when I changed the data.json, it should be working since the watching task has been done.

After searching for a while, I found out that actually node.js require() function has cache, that’s the reason why it won’t update the content even the data.json is changed.

Hey! Where is my media queries?

It’s a common sense that when we build e-mail, we should inline the css into html element tags, just for compatibility and for your own safety. My brilliant colleague has done this for me by including gulp-inline-css in the task, meanwhile, since being responsive is one of the requirements from clients, I put the style sheet for mobile device within <head></head> section. Now everything looks perfect and ready to go, until I found the output page isn’t responsive, oh my god…

What happened to my code?

When I opened the inspector to check head section, I couldn’t find media queries, then I realized that it’s because of gulp-inline-css, I felt stupid for trying to find it, so my question became Can we inline the media queries?, well, the answer is NO Check this.
So we need to find out is there any option in gulp-css-inline, which can help us to solve this issue.

Solution

Thanks to the package builder, there is a option options.preserveMediaQueries, which can prevent the task to inline the media queries,
